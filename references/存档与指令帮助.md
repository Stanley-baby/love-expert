# 存档功能帮助（给恋爱专家查询用）

## 一、整体流程
1. **检查快捷加载** `/love-load` 或 `#@memory load`
2. **读取索引** `memory/<user_id>/index.json`
   - 有对象 → 询问是否选用 → 载入 target.json
   - 无对象 / 索引缺失 → 询问 user_id & target_id（缺省 `default` / `target-<timestamp>`）→ 自动建档
3. **进入技能核心**（咨询/学习/话术）
4. **会话中/结束** 如触发 `#@memory save` / `/love-save` → 写入 target.json + index，并在缺省时自动生成 6-10 行详细摘要
5. 若用户声明“只学习”，保持无对象模式

## 二、数据结构
- `memory/<user_id>/index.json`
  ```json
  {
    "user_id": "default",
    "targets": [
      {
        "target_id": "小A",
        "alias": "小A",
        "summary": "上次聊到她对约会时间还在犹豫",
        "last_updated": "2026-03-16T10:00:00+08:00"
      }
    ]
  }
  ```
- `memory/<user_id>/<target_id>.json`
  ```json
  {
    "user_id": "default",
    "target_id": "小A",
    "profile": { "关系来源": "相亲", "性格": "慢热" },
    "stage": "暧昧初期",
    "history": ["2026-03-15 约了周六咖啡，对方说不确定"],
    "last_summary": "……",
    "updated_at": "2026-03-16T10:00:00+08:00"
  }
  ```

## 三、指令速查
| 指令 | 用途 | 语法示例 | 备注 |
| --- | --- | --- | --- |
| `/love <内容>` | 普通学习/咨询 | `/love 怎么判断表白时机` | 无对象 → 不建档 |
| `/love =<target>-<context>` | 快捷咨询（不写存档） | `/love =小A-她今晚约我` | 若无该对象，按规则创建后只加载上下文 |
| `/love-load …` / `#@memory load …` | 强制加载对象上下文 | `/love-load user=default target=小A` | 未传 user → default；若无存档会询问并新建 |
| `/love-save …` / `#@memory save …` | 写入存档 | `/love-save user=default target=小A` | 未传 summary → 自动生成详细摘要并更新 history/index |
| `/love-save =<target>-<event>` | 事件简写存档 | `/love-save =小A-记录周末咖啡邀约` | 当作 user=default；追加 event 到 history |

## 四、自动摘要模板
当 save 未提供 summary 时，使用以下结构生成 6-10 行文字：
1. 关系阶段
2. 关键事实
3. 对方反馈（IOI/IOD、情绪、节奏）
4. 当前问题
5. 下一步动作
6. 风险与雷区
7. 待确认信息

示例：
```
关系阶段：暧昧初期，互动稳定但未线下见面。
关键事实：对方表达好感但以工作忙推迟见面。
对方反馈：回复速度变慢但仍会主动回话。
当前问题：担心冷掉，需要恰当时机再次邀约。
下一步动作：先用低压力邀约（短时咖啡），若拒绝转轻话题维持联系。
风险与雷区：避免连续追问、避免情绪化施压。
待确认信息：对方近两周的空闲时间与兴趣点。
```

## 五、无对象学习模式
- 用户只想学习技巧或概念时，可以直接 `/love` 或自然语言提问。
- 技能不会强制建档，除非用户触发 `/love-load`、`/love-save` 或明确提供对象信息。
